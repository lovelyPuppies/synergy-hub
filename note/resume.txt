    ❔ TCN (Train Communication Network)

      1. 개요
        TCN(Train Communication Network)은 철도 차량 내 제어 장치 간 통신을 표준화하기 위한 네트워크 프로토콜이다.
        철도 차량 시스템의 안정적이고 신뢰성 있는 데이터 전송을 위해 IEC 61375 국제 표준을 따르며,
        열차 내 장치 및 차량 간 네트워크 통신을 지원한다.

      2. 주요 네트워크 구성 요소
        - MVB (Multifunction Vehicle Bus)
          차량 내부 통신을 담당하는 네트워크
          필드버스 기반의 주기적 데이터 전송 방식 사용
          정적인 주소 할당 방식 적용

        - WTB (Wire Train Bus)
          차량 간 통신을 담당하는 네트워크
          동적인 주소 할당 방식 적용
          차량 추가 시 자동으로 네트워크 확장이 가능

        - Ethernet
          고속 데이터 전송이 필요한 경우 사용
          WTB 및 MVB와 함께 사용되어 시스템 통합

      3. 데이터 전송 방식
        - 주기적 데이터 전송 (Periodic Transmission)
          필드버스 기반으로 차량 내 장치 간 데이터를 정기적으로 송수신
          철도 시스템의 안정성과 실시간성을 보장

      4. 주소 할당 방식
        - MVB: 정적인 주소 할당 (Static Addressing)
        - WTB: 동적인 주소 할당 (Dynamic Addressing)
          차량이 추가될 경우 자동으로 네트워크에 통합됨

      5. 확장성
        WTB를 사용하면 차량 추가가 가능하여 네트워크 확장이 용이함

      6. 표준화 기관
        - IEC 61375 (국제 표준)

      7. 기능 안전 표준
        - IEC 61375 + EN 50128
          철도 시스템의 안전성과 신뢰성을 보장하기 위한 기능 안전 표준 적용

      8. 네트워크 계층 구조
        - MVB (차량 내부 장치 간 통신)
        - WTB (차량 간 통신)
        - Ethernet (고속 데이터 통신)

      TCN은 철도 시스템의 특성에 맞게 안정적인 데이터 전송을 보장하며,
      열차 제어 및 운영의 핵심 역할을 수행한다.

    ❔ 왜 cyclic redundancy check (CRC) 보다 forward error correction (FEC) 더 널리 사용될까?
      * CRC vs. FEC, 핵심 차이점
        - CRC: 오류를 검출하지만 수정 불가, 재전송(ARQ) 필요
        - FEC: 재전송 없이 수신 측에서 오류 자동 수정 가능
      * 왜 FEC가 더 널리 사용될까?
        - 재전송 문제 해결 (저지연 & 안정성 확보)
          - CRC는 오류 검출 후 재전송(ARQ) 필요 → 지연 증가, 네트워크 혼잡
          - FEC는 재전송 없이 즉시 오류 수정 → 위성통신, 5G, 스트리밍, IoT 등 필수
        - 고속`·대용량 데이터에 적합
          - 5G, 광통신, Wi-Fi, SSD 등에서는 빠른 데이터 처리가 필수
          - LDPC, Turbo, Polar Codes 등 FEC 기술이 핵심
        - 연결이 불안정한 환경에서 필수
          우주`, 해저, 위성, 무선통신 등 재전송이 어렵거나 불가능한 상황에서 FEC가 안정적인 데이터 전송 보장

    SOOP이 추구하는 “민주적인 미디어 플랫폼”과 글로벌 서비스 통합 비전은 기술을 통해 사용자와 스트리머를 연결하고, 콘텐츠 생태계를 혁신한다는 점에서 깊은 인상을 받았습니다. 특히, 생성형 AI와 자동 번역 기술, 글로벌 스트리밍 계정 연동 등 차세대 기술 도입을 통해 사용자 편의성과 플랫폼의 확장성을 강화하려는 노력에 매료되었습니다.

    저는 C/C++ 기반 프로그래밍과 ❗ 멀티미디어 데이터 처리 기술에 대한 경험을 쌓아왔으며, 비동기 프로그래밍 이론과 소켓 프로그래밍 지식을 활용한 네트워크 시스템 설계 역량을 보유하고 있습니다. 이러한 역량을 기반으로, SOOP의 글로벌 스트리밍 플랫폼 개발에 기여하며 새로운 콘텐츠 생태계를 구축하는 데 동참하고 싶습니다.

    입사 초기에는 SOOP의 기존 기술 스택과 시스템 아키텍처를 깊이 이해하며, 플랫폼의 안정성을 확보하고 최적화하는 데 집중하겠습니다. 이후, 글로벌 사용자들에게 더 나은 경험을 제공할 수 있는 실시간 미디어 처리 기술과 AI 기반 서비스의 개발에 참여하겠습니다.

    특히, SOOP에서 도입 예정인 AI 스트리밍 매니저(SARSA)나 생성형 AI 기술(SAVVY)와 같은 혁신적인 기술 개발 프로젝트에 기여하여, 스트리머와 사용자가 더 쉽게 소통하고 콘텐츠를 창출할 수 있는 환경을 제공하고 싶습니다. 이를 통해, SOOP이 글로벌 미디어 시장의 리더로 성장하는 데 핵심적인 역할을 하겠습니다.


  성장 과정
    "기술 활용의 범위를 확대해온 탐구의 여정"

    어릴 적부터 기술을 활용해 문제를 해결하고, 사람들이 원하는 환경을 제공하는 데 흥미를 느껴왔습니다. 마인크래프트 서버를 운영하며 사용자들의 요구를 분석하고 다양한 모드와 설정을 조합해 맞춤형 환경을 제공한 경험은 기술을 통해 사람들의 필요를 채우고 즐거움을 줄 수 있다는 것을 깨닫게 해주었습니다. 이러한 경험은 사용자 중심적 관점을 갖추게 했으며, 컴퓨터 공학을 전공으로 선택하게 된 계기가 되었습니다.

    대학교 졸업 후 IT 회사에서 웹 개발자로 활동하며 사용자 데이터를 기반으로 효율적인 서비스를 설계하고 운영하는 업무를 맡았습니다. 예를 들어, Django와 Flask를 활용해 위협 감지 시스템의 REST API를 설계하고, 고객 동선 분석 시스템에서 매장 사진 매핑 알고리즘을 구현한 경험이 있습니다. 또한, 연구 프로젝트에서는 TensorFlow를 활용해 카페 음식 및 음료를 판별하는 모델을 개발하며 데이터 처리와 모델 학습 과정을 직접 수행했습니다. 이러한 경험을 통해 다양한 기술 스택을 다루며 문제 해결 역량과 개발자로서의 전문성을 쌓을 수 있었습니다.

    이후 "[Intel] 엣지 AI SW 아카데미 (교육)"에서 임베디드 시스템과 AI 기술을 학습하며, 소프트웨어와 하드웨어를 통합하여 문제를 해결하는 기술을 익혔습니다. STM32, 소켓 통신, 커널 모듈 개발, Qt 등 다양한 기술을 다루며 새로운 분야에 대한 도전과 학습의 즐거움을 느꼈고, 이를 통해 효율적인 시스템 설계 역량을 키웠습니다.

    이러한 경험을 통해 기술적 문제를 해결하는 역량과 사용자 중심적 시각을 갖추게 되었습니다. 앞으로도 이러한 강점을 바탕으로 조직의 목표에 기여하며 성장하고자 합니다.


  성격 및 장단점
    "꼼꼼함으로 완성도를 높이는 문제 해결사"
    
    일을 할 때 항상 세부적인 부분까지 검증하며 완성도를 높이는 데 집중합니다. 단순히 주어진 과제를 해결하는 데 그치지 않고, 과정에서 발생할 수 있는 오류나 개선점을 미리 파악해 보완하는 데 신중합니다.

    예를 들어, Python 개발 과정에서 pytest를 활용해 모듈별 유닛 테스트를 작성하고, Jupyter Notebook에서 짧은 코드 블록을 빠르게 검증하며 기능 구현의 정확성을 확보한 경험이 있습니다. 또한, 코드 유지보수의 편리성을 위해 주요 함수와 모듈에 docstring을 작성하여 동료들이 쉽게 이해할 수 있도록 문서화를 체계적으로 관리했습니다. 개인 프로젝트에서는 monorepo 구조를 활용해 코드를 일관되게 관리하고, 변경 사항을 신속히 테스트할 수 있는 환경을 구축하여 작업 효율성을 높인 바 있습니다. 이러한 꼼꼼함 덕분에 팀원들로부터 "믿고 맡길 수 있다"는 평가를 받으며 협업 과정에서 안정감을 제공할 수 있었습니다.

    하지만 완벽을 추구하는 성향이 때로는 일정 관리의 어려움으로 이어지기도 했습니다. 이를 극복하기 위해 우선순위를 명확히 설정하고, 기한 내 최적의 결과를 내기 위해 최선의 방안을 고민하고 실행하려 노력하고 있습니다. 최근에는 이러한 노력을 통해 기한을 준수하면서도 높은 품질을 유지하는 성과를 낸 경험도 있습니다.

    앞으로도 꼼꼼함을 통해 결과물의 신뢰성과 완성도를 확보하며, 효율적인 문제 해결을 통해 팀과 조직의 목표를 달성하는 데 기여하고자 합니다. 이를 위해 명확한 우선순위를 설정하고, 변화에 유연하게 대처하는 자세를 유지하며 꾸준히 노력하겠습니다.


  특기 사항 📅 Reviewed at 2025-01-21 15:45:30
    "개발 환경 최적화로 성과를 높이는 엔지니어"

    저는 개발 생산성을 높이고, 운영 중 발생하는 문제를 신속히 해결하며, 효율적인 개발 환경을 구축하는 데 주력해 왔습니다.

    예를 들어, Fish와 PowerShell 스크립트를 활용해 환경 초기화 및 네트워크 설정 작업을 자동화한 경험이 있습니다. 이를 통해 반복적이고 시간이 소요되는 작업을 단축하며, 작업 오류를 줄이고 신뢰도를 향상시켰습니다. 이러한 자동화는 팀의 개발 생산성을 높이는 데 크게 기여했습니다.

    또한, Docker와 DevContainer를 사용하여 독립적인 프로토타입 환경을 구축한 경험도 있습니다. 이를 통해 여러 팀원이 동일한 개발 환경에서 작업할 수 있는 기반을 제공하며, 협업과 프로토타입 개발 속도를 향상시켰습니다. 특히, ROS 기반 프로젝트에서 Docker를 활용하여 네트워크 연결과 시스템 설정을 간소화한 환경을 구현하며 프로젝트 성공에 기여했습니다.

    운영 중 발생한 문제를 해결한 경험도 다수 보유하고 있습니다. Fish Shell 환경에서 발생한 sudo 캐시가 서브프로세스에서 무효화되는 문제를 분석하고 환경 변수 설정 방안을 제안해 문제를 해결한 사례가 있으며, Jetson Nano 에서 OpenCV 에서 cv2.imshow 사용 시 발생하는 Jupyter Notebook의 충돌 문제를 해결하기 위해 대체 솔루션을 제안하였습니다.

    이처럼 저는 문제 해결 능력과 생산성 향상을 기반으로 한 경험을 통해, 개발팀과 조직에 실질적인 가치를 제공하는 데 집중하고 있습니다. 이를 바탕으로 SOOP에서 보다 효율적이고 신뢰할 수 있는 개발 환경을 만들어가고 싶습니다.
